    Chart.js • Visualisation de données

    Origine

    Documentation issue d'un contexte client avec des exigences de visualisation. Chart.js est la librairie de référence pour les graphiques web · légère, performante, intégrable dans Angular ou React sans dépendance lourde.

    Installation

    # Chart.js seul
npm install chart.js

# Avec le wrapper Angular (ng2-charts)
npm install chart.js ng2-charts

    Types de graphiques

    Type Cas d'usage
    line Évolution temporelle, métriques
    bar Comparaison de valeurs
    doughnut / pie Répartition en pourcentage
    radar Comparaison multi-axes
    scatter Corrélation entre variables
    bubble 3 dimensions (x, y, taille)

    Utilisation vanilla (sans framework)

    import { Chart, registerables } from 'chart.js';
Chart.register(...registerables);

const ctx = document.getElementById('myChart') as HTMLCanvasElement;

const chart = new Chart(ctx, {
  type: 'line',
  data: {
    labels: ['Jan', 'Fév', 'Mar', 'Avr', 'Mai'],
    datasets: [{
      label: 'Ventes',
      data: [120, 190, 150, 210, 180],
      borderColor: '#38bdf8',
      backgroundColor: 'rgba(56,189,248,0.1)',
      tension: 0.4,    // courbe lissée
      fill: true,
    }]
  },
  options: {
    responsive: true,
    plugins: {
      legend: { position: 'top' },
      title: { display: true, text: 'Évolution des ventes' }
    }
  }
});

    Intégration Angular (ng2-charts)

    Configuration

    // app.config.ts
import { provideCharts, withDefaultRegisterables } from 'ng2-charts';

export const appConfig: ApplicationConfig = {
  providers: [
    provideCharts(withDefaultRegisterables())
  ]
};

    Composant standalone

    import { Component } from '@angular/core';
import { BaseChartDirective } from 'ng2-charts';
import { ChartConfiguration, ChartData } from 'chart.js';

@Component({
  standalone: true,
  imports: [BaseChartDirective],
  template: `
    <canvas baseChart
      [data]="chartData"
      [options]="chartOptions"
      [type]="'line'">
    </canvas>
  `
})
export class SalesChartComponent {

  chartData: ChartData<'line'> = {
    labels: ['Jan', 'Fév', 'Mar', 'Avr', 'Mai'],
    datasets: [{
      data: [120, 190, 150, 210, 180],
      label: 'Ventes',
      borderColor: '#38bdf8',
      tension: 0.4,
      fill: false,
    }]
  };

  chartOptions: ChartConfiguration['options'] = {
    responsive: true,
    scales: {
      y: { beginAtZero: true }
    }
  };
}

    Mise à jour dynamique des données

    // Ajouter un point
addDataPoint(label: string, value: number) {
  this.chartData.labels?.push(label);
  this.chartData.datasets[0].data.push(value);
  this.chart?.update();   // déclenche le re-render
}

// Remplacer toutes les données
updateChart(newData: number[]) {
  this.chartData = {
    ...this.chartData,
    datasets: [{ ...this.chartData.datasets[0], data: newData }]
  };
}

    Donut avec total au centre (plugin custom)

    const centerTextPlugin = {
  id: 'centerText',
  afterDraw(chart: Chart) {
    const { ctx, chartArea: { width, height } } = chart;
    ctx.save();
    ctx.font = 'bold 24px JetBrains Mono';
    ctx.fillStyle = '#e2e8f0';
    ctx.textAlign = 'center';
    ctx.fillText('78%', width / 2, height / 2 + 8);
    ctx.restore();
  }
};

    Tree-shaking · alléger le bundle

    // Au lieu de registerables (tout importer)
import { Chart } from 'chart.js';
import { LineController, LineElement, PointElement, LinearScale, CategoryScale } from 'chart.js';

// Enregistrer seulement ce dont on a besoin
Chart.register(LineController, LineElement, PointElement, LinearScale, CategoryScale);

    Cas concrets

    Chart.js vs D3.js · lequel choisir ?

    Chart.js : déclaratif, rapide à mettre en place, idéal pour les graphiques standards (line, bar, pie). D3.js : impératif, courbe d'apprentissage élevée, pour des visualisations complexes et sur mesure. En mission entreprise : Chart.js couvre 90% des besoins.

    Pourquoi le graphique ne se met pas à jour après changement de données ?

    Chart.js ne détecte pas les mutations directes. Il faut soit appeler chart.update() manuellement, soit créer une nouvelle référence de l'objet data (immuabilité). Avec ng2-charts, réassigner chartData déclenche la mise à jour.

    Comment exporter un graphique en PNG ? 
    const base64 = chart.toBase64Image();
const link = document.createElement('a');
link.download = 'chart.png';
link.href = base64;
link.click();
    Comment gérer de grands volumes de données ?

    Activer le decimation plugin de Chart.js pour réduire automatiquement le nombre de points affichés sans perdre la forme générale de la courbe. Utile dès 1000+ points.

    Points clés

    • Chart.js est déclaratif et couvre 90% des cas en entreprise
    • ng2-charts = wrapper Angular officiel avec typage TypeScript complet
    • toujours tree-shaker les imports pour limiter l'impact sur le bundle
    • chart.update() ou nouvelle référence data pour le re-render
    • responsive + couleurs adaptées au thème = indispensable en dark mode
