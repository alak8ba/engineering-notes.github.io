    kore-backup • Snapshots Chiffrés du VPS

    Disponible sur GitHub

    Sauvegardes incrémentales chiffrées du VPS KORE : restic pour les snapshots déduplités, rclone pour le transfert vers stockage distant, planification cron, drill de restauration documenté.

    Contexte

    Un VPS qui héberge plusieurs services en production sans stratégie de sauvegarde testée, c'est une bombe à retardement. Le scénario du « tout perdre » doit pouvoir être annulé en moins d'une heure.

    kore-backup met en place le pattern standard : snapshots incrémentaux chiffrés, stockage distant, drill de restauration régulier.

    Philosophie

    • Chiffrement client-side : la clé reste sur le VPS, le stockage distant ne voit que du chiffré
    • Déduplication : restic ne re-stocke pas les blocs déjà présents
    • Retention configurable : politique de rétention via forget --prune
    • Drill obligatoire : restauration testée régulièrement, pas seulement les sauvegardes

    Ce que fournit le socle

    Composant Rôle
    restic Snapshots incrémentaux chiffrés, déduplication, vérification d'intégrité
    rclone Transfert vers stockage distant (S3, B2, GCS, OneDrive...)
    Cron Planification quotidienne/hebdomadaire des snapshots
    Scripts de drill Restauration test documentée, à exécuter périodiquement
    .env.example Mot de passe restic, credentials rclone, retention policy

    Cycle de vie

    VPS  →  restic snapshot  →  chiffrement client-side  →  rclone push  →  stockage distant
                                                                              │
                                                                              ▼
                                                                        DRILL régulier
                                                                              │
                                                                              ▼
                                                                  rclone pull → restic restore
