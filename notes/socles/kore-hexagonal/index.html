    kore-hexagonal - Vue d'ensemble

    Contexte

    La majorité des projets Java démarrent avec les mêmes problèmes : structure ad hoc, couplage fort entre le métier et le framework, absence de séparation commande/requête, tests difficiles à écrire.

    kore-hexagonal est extrait de 5 mois de travail réel sur une plateforme SaaS en production. Ce n'est pas un exemple construit pour illustrer un pattern - c'est une base tirée de code qui a tourné avec de vraies données.

    Objectifs

    • Structure hexagonale propre, prête à recevoir n'importe quel domaine métier
    • Séparation commandes et requêtes (CQRS)
    • Communication asynchrone via Kafka intégrée dès le départ
    • Sécurité JWT opérationnelle sans configuration supplémentaire
    • Observable dès le premier démarrage (Prometheus, Grafana)

    Domaine de démonstration : Booking

    Le cas d'usage de démonstration est le domaine Booking : réservation d'une ressource identifiée par assetRef, sur une période définie par startDate et endDate, avec cycle de vie géré par statut.

    Ce domaine est intentionnellement générique. Il peut être remplacé par n'importe quel aggregate métier : commande, contrat, intervention, incident.

    Acteurs

    Acteur Rôle
    Gestionnaire Crée et gère les bookings, valide, clôture
    Opérateur Suit l'exécution des bookings en cours
    Client Consulte les bookings disponibles
    Système Tiers Soumet des demandes via l'API

    Stack technique

    Couche Technologie
    Langage Java 21
    Framework Spring Boot 3.3
    Persistance PostgreSQL + Spring Data JPA
    Messagerie Kafka
    Sécurité Spring Security + JWT stateless
    Observabilité Micrometer, Prometheus, Grafana
    Tests JUnit 5, Mockito, Testcontainers, RestAssured
    CI/CD GitHub Actions + Docker + GHCR

    Valeur ajoutée

    • Démarrage immédiat sans choix d'architecture à faire
    • Domaine isolé, testable sans infrastructure
    • Lecture séparée de l'écriture, projections optimisées via Kafka
    • CI/CD, Docker Compose, monitoring inclus

    Evolution long terme

    kore-hexagonal est la première brique d'un écosystème de socles :

    • kore-batch : traitement par lots, scheduling, partitionnement parallèle
    • kore-stream : ingestion temps réel, Kafka Streams
    • kore-react : frontend React structuré, composants métier réutilisables
